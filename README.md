# Aplication Tracking System for HR (ATS)

Современные HR-отделы обрабатывают большое количество резюме — ручная проверка занимает много времени, субъективна и подвержена ошибкам. Этот проект автоматизирует рутинный скрининг, извлекая структуру из резюме, сопоставляя кандидатов с требованиями вакансий и формируя ранжированный список наиболее подходящих кандидатов.  
По внутренним оценкам, автоматизация сокращает операционные расходы HR на **~40–50%** за счёт ускорения первичного отбора и параллельной обработки.

---

## Ключевые возможности

- **Парсинг резюме**:  
  Извлечение текста из PDF (PyPDF2), очистка и преобразование в структурированный JSON.
- **Настраиваемые критерии**:  
  Гибкая настройка требований (роль, навыки, опыт, образование, знание языков) для точной фильтрации.
- **Параллельная обработка**:  
  Многопоточность для одновременной обработки множества файлов (ускорение до 5x на больших выборках).
- **Ранжирование кандидатов**:  
  Алгоритм сравнения данных резюме с требованиями вакансии и выдача упорядоченного списка.
- **Хранение данных**:  
  Надёжное хранение в PostgreSQL с поддержкой сложных запросов.
- **Интерактивный UI**:  
  Динамические формы, отслеживание прогресса загрузки и отзывчивый интерфейс (JavaScript + AJAX).
- **NER (Named Entity Recognition)**:  
  spaCy для извлечения имён, навыков, языков и контактной информации.
- **Суммаризация**:  
  Модель FRED-T5 для сокращения секций с опытом работы и образованием (до 70% сокращения текста).
- **Семантическое сопоставление **:  
  Испоьзование BART. Дальнейшая интеграция LLM для точного соответствия «вакансия ↔ резюме».

##  Модули (логический обзор)

| Модуль | Описание |
|--------|----------|
| **Ввод критериев** | Динамическая форма для задания требований к вакансии (роль, навыки, опыт и т.д.) |
| **Парсинг резюме** | Извлечение текста из PDF, очистка и преобразование в структурированный JSON |
| **Обработка данных** | Нормализация текста, NER, суммаризация, очистка от шума |
| **Ранжирование** | Алгоритм сравнения профиля резюме с критериями и вычисление рейтинга |
| **Дашборд кандидата** | Фильтруемая страница с превью резюме, рейтингом и деталями |
| **API** | Эндпойнты для загрузки файлов, запуска обработки и получения результатов |

## Стек технологий

| Категория | Технологии |
|-----------|------------|
| **Бэкенд** | FastAPI, Pydantic, Threading |
| **NLP и обработка текста** | spaCy, Natasha, re, FRED-T5 (RussianNLP/FRED-T5-Summarizer), BART (facebook/bart-large-mnli) |
| **PDF-обработка** | PyPDF2 |
| **База данных** | PostgreSQL, psycopg2 |
| **Фронтенд** | JavaScript, AJAX |
